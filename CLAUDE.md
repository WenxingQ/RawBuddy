# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

RawBuddy is a **Photoshop UXP panel plugin** that accepts natural language photo editing commands and applies them as non-destructive Camera Raw / Photoshop adjustments. It uses the Claude API (tool_use) to translate commands into structured adjustment values. No generative AI features are ever applied ‚Äî images remain eligible for photo competitions.

## Commands

All commands run from the `plugin/` directory:

```bash
cd plugin
npm install          # first time only
npm run build        # production build ‚Üí built files emitted to plugin/ root
npm run dev          # development build (unminified)
npm run watch        # watch mode for active development
```

npm scripts invoke webpack via `node node_modules/webpack/bin/webpack.js` directly (not the `webpack` bin shim) because Node.js may not be in the shell PATH on Windows.

## Loading into Photoshop

Use the **Adobe UXP Developer Tool** (install from Creative Cloud):
1. Build: `npm run build`
2. UXP Developer Tool ‚Üí **Add Plugin** ‚Üí select `plugin/manifest.json`
3. Click **Load** next to the plugin entry
4. In Photoshop: **Window ‚Üí RawBuddy**
5. Go to **Settings** tab ‚Üí enter Anthropic API key ‚Üí **Save Key**

After code changes: `npm run build`, then click **Reload** in UXP Developer Tool.

To enable the UXP Developer Tool in Photoshop first: **Edit ‚Üí Preferences ‚Üí Plugins ‚Üí Enable Developer Mode**, then restart Photoshop.

### Data flow

1. User types a command in `CommandInput`
2. `photoshop.js:captureDocumentImage()` captures a downsampled JPEG of the active document (long edge ‚â§ 1024px, quality 0.7) using `ps.imaging.getPixels()` + `encodeImageData()`. Returns `null` gracefully if the imaging API is unavailable (PS < 27.2) ‚Äî the rest of the flow proceeds text-only.
3. `claude.js` POSTs to `https://api.anthropic.com/v1/messages` with `tool_choice: { type: 'tool', name: 'apply_photo_edits' }`. When an image is available the user message content is an array `[{ type: 'image', ... }, { type: 'text', ... }]`; otherwise it is a plain string.
4. Claude visually analyses the image (tones, color casts, subject, mood) and uses that as baseline context, then calls `apply_photo_edits` ‚Üí returns `{ explanation, camera_raw?, photoshop? }`
5. `photoshop.js` applies the edits via native PS adjustment layers:
   - `camera_raw.exposure/contrast` ‚Üí Brightness/Contrast layer (exposure scaled: 1 stop ‚âà 40 brightness units)
   - `camera_raw.highlights/shadows/whites/blacks` ‚Üí Curves layer
   - `camera_raw.vibrance` ‚Üí Vibrance layer
   - `camera_raw.saturation` + `photoshop.hue/saturation_ps/lightness` ‚Üí Hue/Saturation layer
   - `photoshop.brightness/contrast_ps` ‚Üí Brightness/Contrast layer
6. History entry added to React state (session only)

### Hard-won UXP lessons ‚Äî do not revert these

- **webpack externals must use `'commonjs photoshop'`** (not bare `'photoshop'`). With `target: 'web'`, bare string externals resolve as browser globals (`window.photoshop`), which are undefined in UXP. The `commonjs` prefix makes webpack emit `require('photoshop')`.
- **CSS must be inlined in `src/index.html`**, not imported via JS. `style-loader` injects `<style>` tags via DOM APIs that UXP does not support ‚Äî it crashes the panel silently.
- **React must mount via `document.createElement`**, not `document.getElementById`. UXP runs scripts before the HTML body is parsed regardless of `defer` or script placement, so `getElementById` returns null. We create the container at runtime instead.
- **React 17, not 18.** React 18's scheduler uses `MessageChannel` which UXP does not support.
- **`require('photoshop')` and `require('uxp')` must be lazy** (inside functions, not at module top level). Top-level `require` calls in bundled code can fail during module initialisation in UXP.
- **`executeAsModal` may not exist** (`require('photoshop').core` is undefined in some PS versions). `photoshop.js` uses a `withModal()` helper that falls back to a direct call.
- **`window.localStorage`** is used for API key storage. `require('uxp').storage.secureStorage` does not exist in the UXP version bundled with the user's Photoshop.
- **batchPlay values must be plain numbers**, not `{ _unit: 'none', _value: N }` objects.
- **Built files go to the plugin root** (`plugin/index.html`, `plugin/main.js`), not `dist/`. UXP resolves script `src` paths relative to `manifest.json`, not relative to the HTML file ‚Äî so `dist/index.html` loading `<script src="main.js">` would look for `plugin/main.js`, not `plugin/dist/main.js`.
- **Do not pass `modalBehavior` to batchPlay inside `executeAsModal`.** It is only valid outside a modal context; passing it inside causes silent failures or errors.
- **`layer.kind` is a `LayerKind` constant in UXP, not a number.** Comparing `layer.kind !== 3` is always true. Use `ps().constants.LayerKind.SMARTOBJECT` for the comparison.
- **Camera Raw filter (`_obj: 'cameraRaw'`) is not scriptable via batchPlay in PS 2026.** The action event is unrecognised (result -25920). All adjustments use native PS adjustment layers instead.
- **Hue/Saturation batchPlay requires `presetKind` and a nested `adjustment` array.** Without `presetKind: { _enum: 'presetKindType', _value: 'presetKindCustom' }`, the layer is created with default (zero) values and the supplied values are silently ignored. The correct descriptor shape is:
  ```js
  type: {
    _obj: 'hueSaturation',
    presetKind: { _enum: 'presetKindType', _value: 'presetKindCustom' },
    colorize: false,
    adjustment: [{
      _obj: 'hueSaturation',   // same _obj as outer type
      hue: N,
      saturation: N,
      lightness: N,
    }],
  }
  ```
  Flat properties on the type object are ignored. CharID keys (`Strt`, `H   `, `Lght`) cause a -1715 program error.

### Image capture (`captureDocumentImage`)

- **`ps.imaging` is only available in PS 27.2+.** Always guard with `typeof imaging.getPixels !== 'function'` before calling. The function returns `null` on older builds and the flow degrades to text-only automatically.
- **`getPixels` must run inside `withModal`** ‚Äî it composites all visible layers, which PS treats as a document-state operation requiring modal context.
- **Always pass `componentSize: 8` explicitly** to `getPixels`. The default is undocumented and could change in a future PS release, silently breaking JPEG encoding.
- **`encodeImageData` with `mediaType: 'image/jpeg'`** accepts a `quality` float (0‚Äì1). We use `0.7` ‚Äî sufficient for visual analysis, keeps payload to ~80‚Äì150KB base64.
- **Do not call `imageData.dispose()`** after passing it to `encodeImageData`. The UXP imaging runtime manages the lifecycle automatically; explicit disposal can cause access-after-free errors.
- **Base64 conversion must be chunked** (8192 bytes per iteration). Calling `String.fromCharCode(...buffer)` on a large Uint8Array in a single spread will exhaust the JS call stack.

### Adding new adjustment parameters

1. Add to tool schema in `src/prompts/system.js` (both `APPLY_EDITS_TOOL.input_schema` and `SYSTEM_PROMPT` text)
2. Add mapping in `applyEdits` in `src/services/photoshop.js` to the appropriate `make*` helper
3. If a new adjustment layer type is needed, add a `make*` helper following the existing patterns and add `assertOk` result checking

### Model

`claude-sonnet-4-6` with `cache_control: ephemeral` on the system prompt. Prompt caching reduces latency and cost for repeated commands in the same session.
